# Анализ применения машинного обучения в промышленности для прогнозирования ресурсов

## Установка
Для базовой работы проекта (вычисления на процессоре/CPU) достаточно установить стандартные зависимости:
```bash
pip install -r requirements.txt
```
### Опционально (Аппаратное ускорение на GPU):
Поскольку нейросетевые архитектуры требуют значительных вычислительных мощностей, для обучения моделей рекомендуется использовать видеокарту NVIDIA. Для этого установите PyTorch с поддержкой CUDA (версия 12.1):
```Bash
pip install torch torchvision torchaudio --index-url [https://download.pytorch.org/whl/cu121](https://download.pytorch.org/whl/cu121)```
```

## Реализованные архитектуры
В рамках вычислительного эксперимента реализованы и сопоставлены следующие модели:
1. **SARIMA (Seasonal ARIMA):** Классический статистический базис с автоматическим подбором гиперпараметров (Grid Search).
2. **DLinear:** Современная линейная модель с предварительной декомпозицией ряда на тренд и сезонность.
3. **Autoformer:** Трансформерная архитектура с механизмом автокорреляции.
4. **ARIMA-LSTM (Hybrid):** Композитная архитектура, моделирующая линейный тренд статистически, а нелинейные остатки — рекуррентной сетью с учетом экзогенных циклических признаков времени.

## Структура проекта
* `run_pipeline.py` — Автоматически запускает весь цикл: от скачивания данных до итогового сравнения.
* `config.py` — единый пульт управления гиперпараметрами и выбором активного датасета.
* `utils.py` — вспомогательный модуль: фиксация seed, расчет метрик (MSE, RMSE, MAE, MASE, sMAPE, R^2), генерация экзогенных признаков времени.
* `models.py` — общие классы нейросетевых слоев.
* `sarima_baseline.py` / `sarima_grid_search.py` — обучение базовой модели SARIMA с кэшированием весов.
* `dlinear.py` — обучение линейной модели DLinear.
* `autoformer.py` — обучение архитектуры Autoformer.
* `hybrid_smart_arima_lstm.py` — обучение компенсирующей сети LSTM на остатках ARIMA.
* `predict.py` — модуль инференса: загружает веса всех обученных моделей и генерирует единый сравнительный дашборд.

## Настройка и конфигурация
Всё управление проектом осуществляется через файл `config.py`. Вы можете переключаться между датасетами, просто изменив одну переменную:

```python
# Варианты: 'steel', 'ett'
ACTIVE_DATASET = 'steel'
```
Архитектура папок (веса и графики) создается динамически под каждый датасет (например, `weights_steel/` и `plots_steel/`), что предотвращает перезапись результатов при смене набора данных.

## Запуск проекта
```Bash
python run_pipeline.py
```
## Результаты работы
По завершении работы скрипта в директории `plots_<название_датасета>/` будут сформированы графики для каждой модели в отдельности, а также итоговый дашборд `production_comparative_dashboard.png`, наглядно демонстрирующий разницу в аппроксимирующей способности алгоритмов.
